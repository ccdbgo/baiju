# 白驹信息架构与页面结构

## 1. 信息架构目标

白驹的信息架构需要同时满足三个要求：
- 高频操作快：新增、查看今日安排、完成任务、打卡要足够快。
- 模块独立清晰：每个核心能力单独入口，方便直达。
- 数据能汇总：各模块最终能回收到统一时间线与今日视图。

## 2. 顶层信息架构

### 2.1 一级结构
- 今日
- 日程
- 待办
- 习惯
- 纪念日
- 目标
- 笔记
- 时间线
- 我的 / 设置

### 2.2 顶层设计说明
- `今日` 是默认首页，承担聚合和快速处理职责。
- `日程、待办、习惯、纪念日、目标、笔记` 保持独立入口，降低查找成本。
- `时间线` 用于统一回顾，不与首页混淆。
- `我的 / 设置` 负责个人资料、通知、同步、赞助等低频功能。

## 3. 跨端导航建议

### 3.1 移动端导航
建议采用底部导航 + 顶部二级切换：
- 底部导航：今日、日程、待办、时间线、我的
- 其他模块入口：通过今日页快捷宫格、顶部功能区或“全部功能”页进入

原因：
- 移动端底部导航承载 4 到 5 个高频入口更符合使用习惯。
- 纪念日、目标、笔记、习惯属于高频但不一定高于待办和日程，适合从快捷入口进入。

### 3.2 PC 端导航
建议采用左侧主导航：
- 今日
- 日程
- 待办
- 习惯
- 纪念日
- 目标
- 笔记
- 时间线
- 我的 / 设置

原因：
- PC 端适合承载更多一级导航。
- 左侧导航方便长期使用和快速切换。

## 4. 页面结构设计

### 4.1 今日页
定位：全应用默认首页，承担“查看今天”和“快速处理”的职责。

页面模块：
- 顶部日期区：日期、星期、问候语、天气位（可后置）
- 今日概览区：今日日程数、待办数、习惯完成率、重要提醒
- 快捷新增区：新建日程、新建待办、新建笔记、打卡
- 今日日程区：按时间排序展示
- 今日待办区：优先级或截止时间排序
- 今日习惯区：展示待打卡项
- 纪念日提醒区：展示临近事件
- 今日时间线摘要区：显示今日已完成记录

核心操作：
- 快速新增
- 标记完成
- 打卡
- 延期
- 跳转详情

## 6. 核心用户路径

### 6.1 日常规划路径
打开应用 -> 今日页 -> 查看今日安排 -> 新增待办/日程 -> 完成事项 -> 返回今日页查看进度

### 6.2 自律打卡路径
打开应用 -> 今日页或习惯页 -> 查看待打卡项 -> 完成打卡 -> 同步写入时间线

### 6.3 任务升级路径
待办详情 -> 设置具体时间 -> 转为日程 -> 日程完成 -> 时间线沉淀记录

### 6.4 复盘路径
时间线 -> 选择日期范围 -> 查看日程、完成事项、打卡、笔记 -> 进入目标或笔记详情复盘

## 7. 页面设计原则

- 首页强调聚合，不承载过深编辑逻辑。
- 每个模块保持“列表/总览 -> 详情 -> 编辑”的标准结构。
- 新建和快速完成操作要固定位置，便于形成习惯。
- 详情页要承担跨模块关联能力，例如关联目标、关联笔记、加入时间线。
- 移动端优先单列信息流，PC 端优先多栏效率布局。

## 8. 原型建议

### 8.1 移动端优先原型
建议先输出以下页面：
- 今日页
- 日程页
- 待办页
- 习惯页
- 时间线页
- 新建事项弹层

### 8.2 PC 端优先原型
建议先输出以下页面：
- 左侧导航 + 今日工作台
- 日程周视图
- 待办看板/列表
- 笔记编辑页
- 时间线复盘页
